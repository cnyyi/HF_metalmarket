## 进销存模块需求讨论（已搁置）

### 已确认的需求决策
- **业务场景**：自营+商户双轨制（场景C）
- **过磅关联**：混合模式——自营商品强制关联过磅记录，商户进出货可选手动登记或关联过磅
- **库存模型**：流水制，不做实时库存计算，月底人工盘点
- **财务联动**：强联动——采购入库自动生成应付，销售出库自动生成应收
- **商品管理**：按品种管理（大类），用字典表维护（如螺纹钢/钢板/铜管）
- **商户门户**：暂不涉及进销存，后续再扩展

### 推荐方案（待确认）
- 方案三：两层弹性——出入库单为主，可选挂订单
- 核心单据：入库单（StockIn）+ 出库单（StockOut）
- 可选关联采购合同/销售合同，不强制
- 入库单确认→自动创建应付，出库单确认→自动创建应收

### 状态：搁置，用户说先做别的

## 费用单模块需求讨论（已完成规格文档）

### 需求背景
- 市场存在长期固定支出（垃圾清运、临时用工等），现有Payable无法记录详细信息
- 痛点：缺乏业务关联（同一次业务拆成多笔应付拼不回全貌）、缺乏明细描述（临时工谁/何时/做什么）、缺乏费用归集（按大类统计）

### 已确认决策
- **数据结构**：主从——ExpenseOrder（费用单）+ ExpenseOrderItem（明细行）
- **应付生成**：每条明细行→独立一条Payable，拆分核销
- **工作流**：建单即确认，立即生成应付，无草稿状态
- **导航位置**：市场管理下拉菜单
- **费用大类**：字典表维护（expense_category：垃圾清运/临时用工/维修维护/其他支出）
- **商户门户**：暂不涉及

## 地块列表 — 新增租赁状态列

### 变更
- `/plot/list_data` API 增加 `RentStatus` 字段，通过 ContractPlot+Contract EXISTS 子查询动态计算
  - 租赁中：存在关联合同且 StartDate <= GETDATE() <= EndDate 且 Status != N'已终止'
  - 空闲：不存在上述有效合同
- 新增 `rent_status` 查询参数，支持按租赁状态筛选
- `plot/list.html` 表格新增"租赁状态"列（badge样式：绿色=租赁中，灰色=空闲）
- 筛选区新增"全部租赁状态"下拉框（租赁中/空闲）

### Bug 修复
- **问题**：租赁状态全部显示为空闲，即使存在有效合同
- **根因**：Contract 表的 StartDate/EndDate 是 DATE 类型，GETDATE() 返回 DATETIME，SQL Server 隐式将 DATE 转为 DATETIME（时间部分 00:00:00），导致当天日期比较失败（如 EndDate='2026-04-16' → '2026-04-16 00:00:00' < GETDATE()='2026-04-16 20:11:xx'）
- **修复**：`GETDATE()` → `CAST(GETDATE() AS DATE)`，确保 DATE 与 DATE 比较
