## Bug修复：电表抄表批量保存报错 OUTPUT 语法错误

**问题**：在 `/utility/electricity_meter` 页面填写本月表底并批量保存时，报 SQL Server 语法错误：
```
Incorrect syntax near 'OUTPUT'. (102)
```

**根因**：`utility_service.py` 的 `submit_meter_readings` 方法中，INSERT 语句使用了 `OUTPUT INSERTED.ReadingID` 子句，同时通过 f-string 拼接了 `reading_date_sql` 和 `reading_month_sql`（如 `CAST(N'2026-04-01' AS DATE)`）。SQL Server 的 ODBC 驱动在混合参数化占位符 (`?`)、f-string 拼接表达式和 `OUTPUT` 子句时，无法正确解析语句。

**修复方案**：将 `OUTPUT INSERTED.ReadingID` 改为分两步执行：
1. 先执行纯 INSERT 语句
2. 再通过 `SELECT CAST(SCOPE_IDENTITY() AS INT)` 获取新插入的 ID

**修改文件**：`app/services/utility_service.py` 第 673-691 行
