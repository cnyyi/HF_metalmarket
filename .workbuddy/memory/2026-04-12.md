## 安全修复实施（10项全部完成）

按优先级修复了所有发现的安全和代码质量问题：

### 🔴 严重问题（6项已修复）
1. **数据库密码硬编码** → `config/base.py` 改用环境变量（`DB_SERVER`/`DB_PWD`/`DB_UID`/`SECRET_KEY`），创建 `.env.example` 模板
2. **弱 SECRET_KEY** → 默认值改为 `secrets.token_hex(32)` 自动生成，生产环境强制要求环境变量
3. **路径遍历漏洞** → `uploads` 路由增加 `..`/`/`/`\` 检测；`plot.py` 新增 `_safe_delete_image()` 函数做双重路径校验
4. **SQL注入风险** → `utility_service.py` 新增 `_ALLOWED_TABLES` 白名单 + `_get_table_names()` 映射函数，所有表名通过白名单获取
5. **测试路由暴露** → 移除 `/test`、`/test_upload_page`、`/test_user`、`/test_user_list`、重复 `/user/list`、`/debug_contracts`、`/test_contracts`
6. **print调试语句满屏** → `__init__.py` 全部移除蓝图打印和路由列表打印；`utility.py` create 路由的 ~20 行 print 清理

### 🟡 中等问题（4项已修复）
7. **receivable_routes 无认证** → 添加 `@login_required` 装饰器 + 异常处理
8. **create_merchant 返回 None bug** → 改用 `OUTPUT INSERTED.MerchantID` 获取新 ID 后查询返回
9. **异常信息泄露前端** → 所有路由的 `str(e)` 替换为通用消息，改用 `logger.error()` 记录详细日志（含 exc_info=True 堆栈）
10. **删除操作 GET→POST** → 确认项目已全部使用 POST
