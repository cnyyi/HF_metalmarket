## 垃圾清运模块 Bug 修复（2026-04-20 下午）

### 问题1：列表为空（API 500 错误）
- **根因**：pyodbc 对 SQL Server DATE 类型返回 `str` 而非 `datetime.date`，`garbage_service.py` 中 `row.CollectionDate.strftime('%Y-%m-%d')` 报 `AttributeError: 'str' object has no attribute 'strftime'`
- **修复**：用 `hasattr(x, 'strftime')` 判断，字符串则 `str(x)[:10]`，日期对象则 `.strftime()`
- **影响位置**：`get_collections()` 和 `get_collection_detail()` 两个方法

### 问题2：暗色模式颜色过亮
- **根因**：4个模板（list/add/detail/edit）中部分元素使用了 Bootstrap 原生类（`bg-warning`/`bg-success`/`bg-secondary`/`text-primary`/`alert-info`）和硬编码颜色，暗色模式下对比度过高
- **修复**：为每个模板添加 `[data-theme="dark"]` CSS 覆盖，badge/alert/button-outline 等元素使用半透明背景 + 高亮前景色
- **涉及文件**：`templates/garbage/list.html`、`add.html`、`detail.html`、`edit.html`

### 经验
- pyodbc + SQL Server 的 DATE 列可能返回 `str` 而非 `date` 对象，所有日期字段格式化需做类型兼容处理

## 应付账款批量付款（2026-04-20 下午）

### 需求
按客户显示 Tab 中，操作列增加「付款」按钮，在弹窗中输入金额后，按到期日从远到近逐条抵扣，不足完全抵扣的记为部分付款。

### 修改内容
- **后端**：`FinanceService.batch_pay_by_customer()` 新增方法，在同一数据库事务内：
  1. 查询该客户所有未结清应付（Status IN 未付款/部分付款），按 DueDate ASC 排列
  2. 逐条核销（复用5步联动：PaymentRecord → Payable更新 → 状态更新 → CashFlow → Account余额）
  3. 金额不足完全抵扣的最后一条 → 部分付款
  4. 事务保证：中途出错全部回滚
- **路由**：`/finance/payable/batch_pay` POST API
- **前端**：
  - 按客户 Tab 操作列：未付>0时显示「付款」按钮
  - 批量付款弹窗：客户名称/总未付/付款金额/付款方式/账户/日期/备注
  - **核销预览**：输入金额后实时计算预览，显示每条应付的抵扣情况（应付#/到期日/原未付/本次抵扣/抵扣后/状态）
  - 暗色模式：alert-info 适配

### 核销测试结果（已回滚）
- 付款 ¥3000 → 应付#5(¥2064.70)全部抵扣→已付款 + 应付#6(¥935.30)部分抵扣→部分付款
- 账户余额 15000→12000，正确

### 顺手修复
- `get_payables` 搜索报错 500：`count_query` 缺少 Merchant/Customer 的 LEFT JOIN，导致 WHERE 中 `m.MerchantName` 无法绑定
  - 修复：给 count_query 补上 JOIN，WHERE 改为 `WHERE p.IsActive=1` + `AND 动态条件`

## 垃圾清运编辑 Bug 修复 + Payable 联动更新（2026-04-20 晚）

### Bug：编辑按钮 ID 未传递
- **现象**：列表页编辑按钮跳转 URL 为 `/garbage/edit/0`，collection_id 始终为 0
- **根因**：`list.html` 中 `<a href="{{ url_for('garbage.edit', collection_id=0) }}".replace('/0', '/' + item.collection_id)` 语法错误——`.replace()` 写在了 HTML 属性引号外，JS 解析时实际未执行
- **修复**：改为直接用 JS 模板字符串拼接 URL：`href="/garbage/edit/${item.collection_id}"`，详情按钮同理
- **涉及文件**：`templates/garbage/list.html`

### 功能：编辑保存联动更新 Payable
- **原逻辑**：`update_collection()` 只在状态→已结算时更新 Payable 状态，金额/供应商/日期变更不同步
- **新逻辑**：编辑保存后联动更新关联 Payable：
  - 供应商变更 → 更新 VendorName + CustomerID
  - 日期变更 → 更新 DueDate
  - 描述/类型变更 → 更新 Description
  - 金额变更 → 按状态区分：未付款直接改 Amount/RemainingAmount；部分付款改 Amount 且 RemainingAmount=新总额-已付；已付款不允许改（锁定）
  - 状态→已结算 → 更新 Payable 状态为已付款
- **涉及文件**：`app/services/garbage_service.py`
