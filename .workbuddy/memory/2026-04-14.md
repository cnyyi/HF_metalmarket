## 修复批量保存报错：UtilityReading.MerchantID NOT NULL

**问题**：批量保存时报错 `Cannot insert the value NULL into column 'MerchantID'`，因为 INSERT 语句缺少 MerchantID 字段。

**根因**：`submit_meter_readings()` 查询合同关联表时只取了 `UnitPrice, ContractID`，没有 JOIN Contract 获取 MerchantID，INSERT 时也未包含该字段。

**修复**：
1. `utility_service.py`：查询合同关联表时 JOIN Contract，同时获取 `MerchantID`
2. INSERT 语句增加 `MerchantID` 字段和参数
3. 数据库设计文档补充 `MerchantID INT NOT NULL` 字段说明

**二次修复**：发现 `CreatedBy INT NOT NULL` 字段也缺失，同样导致 NOT NULL 报错。
- 路由层注入 `current_user.id` 到每个 reading 的 `created_by`
- service 层 INSERT 增加 `CreatedBy` 字段
