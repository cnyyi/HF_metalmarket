## Agent 返回0的Bug修复

### 根因
Agent 历史对话中包含了之前的错误结果（`Status = N'执行中'` 和 `N'有效'` 返回0），LLM 看到历史消息后直接复述旧答案，不再生成新的 SQL 查询。

MessageID=8 的 `GeneratedSQL: None` 证明 LLM 根本没生成 SQL，而是从上下文推断。

### 修复
1. 清除了包含错误数据的 Agent 历史对话（ConversationID=1，8条消息）
2. 在 System Prompt 规则中新增第11条：**严禁从历史对话中复用查询结果**，必须重新生成 SQL 查询数据库
