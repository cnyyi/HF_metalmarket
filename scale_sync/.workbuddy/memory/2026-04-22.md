## 过磅同步阶段3补漏机制实现

- **问题**：Access 到 SQL Server 的数据不一致，阶段1增量同步用 `流水号 > last_serial AND RecordFinish=1` 过滤，导致跨天过磅记录（皮重录入时 RecordFinish=0 被跳过，毛重录入后 RecordFinish=1 但流水号已小于 last_serial）永远不会被同步
- **实现**：新增 `fetch_missing_records` 函数作为阶段3
  1. 从 Access 取最近 N 天（`missing_detection_days`，默认7天）内 RecordFinish=1 且有流水号的记录
  2. 从 SQL Server 取已同步的流水号（分批500条查询避免 IN 过长）
  3. 差集 = 缺失的流水号
  4. 从 Access 取缺失记录的完整数据（分批查询）
  5. 插入 SQL Server（重复流水号跳过）
- **首次运行**：补录 36 条缺失记录，第二次运行无重复插入（幂等）
- **配置项**：`missing_detection_days` 已存在于 sync_config.json 中（值为7）
- `do_sync` 汇总日志增加补漏数量

## 应收导出 Excel 报错修复

- **问题**：点击"导出Excel"报 `'MergedCell' object attribute 'value' is read-only`
- **原因**：`export_receivables` 中 `ws.merge_cells('A:D')` 后循环仍对 B/C/D 列写入 value
- **修复**：合并区域只写左上角 A 列的值和样式，B/C/D 列只设样式不赋值
