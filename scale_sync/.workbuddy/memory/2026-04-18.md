## 过磅数据同步 - 变更检测功能

**问题**：Access 过磅软件中已完成的磅单可能被修改，导致同步数据与源数据不一致。

**解决方案**：
- SQL Server ScaleRecord 表新增3个字段：
  - `IsModified` (BIT DEFAULT 0)：标记原记录已被修改过时
  - `ModifiedFromRecordID` (INT NULL)：新记录指向被修改的原记录ID
  - `SourceUpdateTime` (DATETIME NULL)：存储 Access 端的更新时间
- 迁移脚本：`scale_sync/migrations/001_add_modified_tracking.sql`
- 同步逻辑改为两阶段：
  1. 阶段1 - 变更检测：对比最近 N 天已同步记录的 SourceUpdateTime 与 Access 端更新时间
  2. 阶段2 - 增量插入：按流水号继续插入新记录
- 发现修改时：原记录标记 IsModified=1，插入新版本记录（ModifiedFromRecordID 指向原记录）
- 仪表盘统计已加 IsModified=0 过滤，排除过时记录
- 新配置项 `change_detection_days`（默认3天）控制回查范围

**修改的文件**：
- `scale_sync/scale_sync.py` - 核心同步逻辑重写
- `scale_sync/sync_config.json` - 新增 change_detection_days 配置
- `scale_sync/migrations/001_add_modified_tracking.sql` - 新建迁移脚本
- `app/services/scale_service.py` - 查询增加新字段，统计排除已修改记录
