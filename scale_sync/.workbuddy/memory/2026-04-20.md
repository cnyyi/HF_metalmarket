# 2026-04-20

## 过磅同步变更检测逻辑修复

- **问题**：原 `detect_changes` 从 SQL Server 的 `CreateTime` 取最近 N 天记录作为变更检测候选集，导致超过检测窗口的老记录即使被修改也检测不到
- **修复**：反转检测方向——以 Access 端 `更新时间` 为依据，从 Access 查最近 N 天内更新过的记录，再到 SQL Server 找对应记录对比
- **替换函数**：
  - `fetch_recent_source_records` → `fetch_recently_updated_access_records`（从 Access 按 更新时间 过滤）
  - `fetch_access_updated_records` → `fetch_sql_records_by_serials`（从 SQL Server 按流水号查找已同步记录）
- **核心变化**：检测窗口 `change_detection_days` 现在过滤的是 Access 端的更新时间，而非 SQL Server 的 CreateTime

## 过磅同步补漏机制

- **问题**：跨天过磅（皮重19日、毛重20日完成）的记录，增量同步时因流水号已过窗口且当时 RecordFinish=0，永远不会被同步
- **修复**：新增 `fetch_missing_records` 函数，对比 Access 已完成记录与 SQL Server 已同步记录，找出缺失项
- 新增配置项 `missing_detection_days`（默认7天）
- `do_sync` 增加阶段3"补漏"，在增量插入后检查遗漏

## meter.py strftime 报错修复

- **问题**：`InstallationDate` 在 SQL Server 中是 VARCHAR 类型，pyodbc 返回 str，对 str 调用 `.strftime()` 报错
- **修复**：新增 `_format_date()` 和 `_format_datetime()` 安全转换函数，兼容 datetime 对象和字符串
- 替换 WaterMeter、ElectricityMeter 的 get_all 和 get_by_id 中共6处 strftime 调用

## ReceivableDetail 插入 NULL 报错修复

- **问题**：`SCOPE_IDENTITY()` 在 pyodbc 中单独 execute，与 INSERT 不在同一 SQL 批次，ODBC 驱动丢失作用域返回 NULL，导致 ReceivableDetail.ReceivableID 插入 NULL 报错
- **修复**：将 `SELECT SCOPE_IDENTITY()` 改为 `OUTPUT INSERTED.ReceivableID` 附加在 INSERT 语句中，原子获取新 ID

## 收费按钮"处理中..."卡住修复

- **问题**：reading_data 页面点击收费按钮后，按钮显示"处理中..."不恢复，但后端数据已入库（金额入账、资金流水已生成）
- **修复**：
  1. 前端 `confirmPay` AJAX 从 `success/error` 双分支恢复按钮 → 改为 `complete` 回调兜底恢复，确保任何情况下按钮都会恢复
  2. 增加 `timeout: 30000`（30秒超时），避免请求无限等待
  3. 改进 `error` 回调：解析服务端返回的 JSON 错误消息；超时时提示"请检查数据是否已入账后刷新页面"
  4. 后端 `pay_reading` 路由增加请求参数和结果日志，便于排查

## 应收列表搜索报错修复

- **问题**：应收列表输入搜索文字后点击搜索，报"网络错误，请稍后重试"
- **原因**：`receivable_service.py` 的 `get_receivables` 方法中 `row.DueDate.strftime()` 和 `row.CreateTime.strftime()` 对 VARCHAR 列返回的 str 类型报 AttributeError，导致后端500
- **修复**：与 meter.py 一致，新增 `_format_date()`/`_format_datetime()` 安全函数替换 strftime 调用

## 应收导出 Excel 报错修复

- **问题**：点击"导出Excel"报 `'MergedCell' object attribute 'value' is read-only`
- **原因**：`export_receivables` 中第358行 `ws.merge_cells('A:D')` 后，循环仍对 B/C/D 列写入 value；openpyxl 合并单元格中非左上角单元格为 MergedCell，不可赋值
- **修复**：合并区域只写左上角（A列）的值和样式，B/C/D 列只设样式不赋值；E-G 列正常赋值；H-L 列赋空字符串
