# 宏发金属交易市场管理系统 - 数据库设计

## 1. 数据库概述

本系统使用SQL Server数据库，采用关系型数据库设计，确保数据的完整性、一致性和安全性。数据库名为`hf_metalmarket`，包含用户管理、商户管理、地块管理、合同管理、水电计费、财务管理和磅秤管理等模块的表结构。

## 2. 表结构设计

### 2.1 系统字典表（Sys_Dictionary）

用于存储系统中的通用字典数据，如商户类型、费用项目、地块状态、业态类型等，避免代码硬编码。

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| DictID | INT | PRIMARY KEY, IDENTITY(1,1) | 字典ID |
| DictType | NVARCHAR(50) | NOT NULL | 字典类型（如merchant_type、expense_item、plot_status、business_type等） |
| DictCode | NVARCHAR(50) | NOT NULL | 字典编码（唯一标识） |
| DictName | NVARCHAR(100) | NOT NULL | 字典名称（显示名称） |
| Description | NVARCHAR(200) | NULL | 字典描述 |
| SortOrder | INT | DEFAULT 0 | 排序顺序 |
| IsActive | BIT | DEFAULT 1 | 是否有效（1：有效，0：无效） |
| CreateTime | DATETIME | DEFAULT GETDATE() | 创建时间 |
| UpdateTime | DATETIME | NULL | 更新时间 |

**字典类型示例：**
- merchant_type: 商户类型（个体工商户、公司、意向商户、业务往来）
- expense_item: 费用项目（租金、水费、电费、过磅费等）
- plot_status: 地块状态（空闲、已租、维修中）
- business_type: 业态类型（金属材料、五金工具、机械配件等）
- contract_status: 合同状态（有效、已过期、已终止）
- payment_method: 付款方式（现金、转账、支票等）

### 2.16 水电费抄表记录表（UtilityReading）

用于存储水电费抄表记录。

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| ReadingID | INT | PRIMARY KEY, IDENTITY(1,1) | 抄表记录ID |
| MeterID | INT | NOT NULL | 表ID（电表或水表） |
| MeterType | NVARCHAR(50) | NOT NULL | 表类型（electricity：电表，water：水表） |
| LastReading | DECIMAL(10,2) | NOT NULL | 上次表底（自动从上一次抄表记录或换表记录获取） |
| CurrentReading | DECIMAL(10,2) | NOT NULL | 当前表底（手动输入） |
| Usage | DECIMAL(10,2) | NOT NULL | 用量（CurrentReading - LastReading，自动计算） |
| UnitPrice | DECIMAL(10,2) | NOT NULL | 单价（从合同或系统设置获取） |
| TotalAmount | DECIMAL(12,2) | NOT NULL | 总金额（Usage * UnitPrice，自动计算） |
| ReadingDate | DATETIME | NOT NULL, DEFAULT GETDATE() | 抄表日期 |
| ReadingMonth | NVARCHAR(7) | NOT NULL | 抄表月份（格式：YYYY-MM） |
| MerchantID | INT | NOT NULL | 商户ID（从合同关联获取） |
| BelongMonth | NVARCHAR(20) | NULL | 所属月份（格式：YYYY年MM月，用户界面选择） |
| CreateTime | DATETIME | DEFAULT GETDATE() | 创建时间 |

**抄表流程说明：**
1. 系统自动获取该表的上次表底（优先从最近的抄表记录，其次从换表记录）
2. 用户输入当前表底，系统自动计算用量和总金额
3. 保存抄表记录后，当前表底自动成为下次抄表的起始底数
4. 如果发生换表，通过换表记录功能手动设置新表的起始底数

**字段变更说明：**
- ~~ContractID~~ - 合同ID字段已删除，通过合同水电表关联表查询
- ~~MerchantID~~ - 商户ID字段已删除，通过合同水电表关联表查询
- ~~CreatedBy~~ - 抄表人字段已删除

## 5. 数据库维护

### 5.1 定期备份

建议定期备份数据库，确保数据安全。可以使用SQL Server的维护计划功能，设置定期全量备份和差异备份。

### 5.2 索引优化

对于经常查询的表，可以创建适当的索引，提高查询性能。例如：

```sql
-- 为用户表的用户名创建索引
CREATE INDEX IX_User_Username ON [User](Username);

-- 为商户表的商户名称和联系人创建索引
CREATE INDEX IX_Merchant_MerchantName ON Merchant(MerchantName);
CREATE INDEX IX_Merchant_ContactPerson ON Merchant(ContactPerson);

-- 为合同表的合同编号和商户ID创建索引
CREATE INDEX IX_Contract_ContractNumber ON Contract(ContractNumber);
CREATE INDEX IX_Contract_MerchantID ON Contract(MerchantID);
```

### 5.3 数据库安全

- 定期更新数据库密码
- 限制数据库用户的权限
- 启用数据库审计功能
- 定期检查数据库日志，发现异常情况

## 6. 注意事项

1. **字符类型统一使用NVARCHAR** - 所有字符类型字段统一使用 NVARCHAR，支持 Unicode 编码，完美支持中文字符
2. 所有表都包含CreateTime和UpdateTime字段，用于记录数据的创建和更新时间
3. 所有表都包含IsActive或Status字段，用于标识数据的状态
4. 表之间通过外键关联，确保数据的完整性
5. 敏感数据（如密码）使用加密存储
6. 所有数值类型都使用DECIMAL，避免浮点数精度问题
7. 日期时间类型使用DATETIME，确保时间精度
8. SQL查询必须使用参数化查询，禁止拼接SQL
9. 中文值必须使用N前缀，例如：N'正常'、N'有效'
