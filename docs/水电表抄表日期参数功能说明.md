# 水电表抄表日期参数功能说明

## 功能概述

电表抄表和水表抄表页面支持日期参数，系统会自动设置默认抄表日期为当月1号，并支持URL参数传递日期。

## 实现时间
2026-04-04

## 功能特点

### 1. 自动默认日期
- 如果没有提供日期参数，系统自动设置抄表日期为当月1号
- 例如：当前日期是 2026-01-04，默认抄表日期为 2026-01-01

### 2. URL参数支持
支持通过URL参数传递日期，格式：`?date=YYYY-MM-DD`

**示例**：
```
http://127.0.0.1:5000/utility/electricity_meter?date=2026-01-15
http://127.0.0.1:5000/utility/water_meter?date=2026-01-15
```

### 3. 日期验证
- 验证日期格式是否正确（YYYY-MM-DD）
- 验证日期不能晚于今天
- 如果日期无效，自动重置为当月1号

### 4. 边界情况处理
- **月份过渡**：正确处理12月到1月的过渡
- **闰年**：正确处理闰年2月29日
- **年末**：正确处理12月31日到1月1日的过渡
- **无效日期**：自动重置为当月1号

## 技术实现

### 后端实现

#### 电表抄表页面路由
```python
@utility_bp.route('/electricity_meter')
@login_required
def electricity_meter():
    """
    电表抄表页面
    支持URL参数传递日期，格式：?date=2026-01-01
    """
    from datetime import datetime, date
    from flask import request
    
    reading_date = request.args.get('date')
    
    if reading_date:
        try:
            parsed_date = datetime.strptime(reading_date, '%Y-%m-%d').date()
        except ValueError:
            parsed_date = date.today().replace(day=1)
    else:
        parsed_date = date.today().replace(day=1)
    
    return render_template('utility/electricity_meter.html', 
                         default_reading_date=parsed_date.strftime('%Y-%m-%d'))
```

```javascript

    // 日期已由后端设置，无需前端重复设置
    
    // 添加日期验证
    $('#readingDate').on('change', function() {
        const selectedDate = $(this).val();
        if (selectedDate) {
            const date = new Date(selectedDate);
            const today = new Date();

                alert('抄表日期不能晚于今天！');
                const firstDayOfMonth = new Date(today.getFullYear(), today.getMonth(), 1);
                $(this).val(firstDayOfMonth.toISOString().split('T')[0]);
            }
        }
    });

});
```

## 使用场景

### 1. 正常抄表
访问页面，系统自动设置日期为当月1号：
```
http://127.0.0.1:5000/utility/electricity_meter
http://127.0.0.1:5000/utility/water_meter
```

### 2. 指定日期抄表
通过URL参数指定抄表日期：
```
http://127.0.0.1:5000/utility/electricity_meter?date=2026-01-15
http://127.0.0.1:5000/utility/water_meter?date=2026-01-15
```

### 3. 补录历史数据
可以指定过去的日期进行补录：
```
http://127.0.0.1:5000/utility/electricity_meter?date=2025-12-01
```

### 4. 修改日期
在页面上手动修改日期选择器的值

## 数据流程

```
用户访问页面
    ↓
后端检查URL参数
    ↓
有日期参数？
    是 → 解析日期 → 格式正确？
        是 → 使用指定日期
        否 → 使用当月1号
    否 → 使用当月1号
    ↓
渲染页面，设置默认日期
    ↓
前端显示日期选择器
    ↓
用户可修改日期
    ↓
前端验证日期
    ↓
提交抄表数据
    ↓
后端保存到数据库
```

## 注意事项

1. **日期格式**：统一使用 `YYYY-MM-DD` 格式
2. **时区处理**：使用服务器本地时区
3. **数据库保存**：`ReadingDate` 字段保存用户选择的日期
4. **历史数据**：可以补录历史数据，但不能录入未来日期
5. **权限控制**：需要登录后才能访问
